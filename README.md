# Prompt Library — BUS4005 Portfolio

This folder contains a structured **10-prompt AI library** developed for **BUS4005 Assessment 1** at La Trobe University.

The prompts are designed to automate high-frequency workflows in a **retail business environment**, focusing on:

- HR onboarding
- Store operations
- Employee support
- Performance reporting
- Management insights

Each prompt is treated as a **production-ready workflow asset**, not just a standalone instruction.

---

## Overview

This prompt library demonstrates how **AI prompting can be applied in real business workflows** to improve:

- efficiency  
- consistency  
- scalability  
- decision-making support  

All prompts are designed using structured prompt engineering techniques and aligned to a **connected workflow chain**.

---

## Workflow Structure

The prompts are organised into a **10-step workflow**, representing a realistic retail onboarding and operations lifecycle:
P01 → P02 → P03 → P04 → P05 → P06 → P07 → P08 → P09 → P10

### Workflow Breakdown

### 1. HR & Onboarding
- P01 — Welcome email  
- P02 — Induction schedule  
- P03 — Role briefing  
- P04 — Training checklist  
- P05 — Store policy summary  

### 2. Operations & Employee Support
- P06 — Shift & roster briefing  
- P07 — 30-day feedback template  
- P08 — Employee FAQ assistant  

### 3. Offboarding & Reporting
- P09 — Exit interview summary  
- P10 — Monthly HR insights report  

Each prompt is designed to fit into a **specific step in the workflow chain**, ensuring continuity and practical application.

---

## Prompt Design Approach

All prompts follow structured prompt engineering principles to ensure reliability and usability:

### 1. Role Framing
Each prompt assigns a clear professional role  
(e.g., HR onboarding specialist, HR analyst)

### 2. Action Clarity
Each prompt defines a single, focused task  
to reduce ambiguity and improve output quality

### 3. Context Injection
Inputs are provided using placeholders  
(e.g., `[JOB_TITLE]`, `[STORE_NAME]`)

### 4. Output Control
Outputs are controlled using:
- formatting instructions (tables, bullet points)
- tone guidance
- word limits
- explicit exclusions

### 5. Prompt Chaining
Prompts are designed to work together as a **connected workflow**, rather than isolated tasks.

## What Each Prompt Includes

Each prompt file is documented using a consistent structure:

- **Prompt Text**  
  The exact instruction used in the AI system  

- **Workflow Position**  
  Where the prompt fits in the process  

- **Problem Being Solved**  
  The operational challenge addressed  

- **Automation Potential**  
  Estimated efficiency gains and scalability  

- **Risks & Mitigations**  
  Including hallucination, data quality, and privacy considerations  

- **Version History**  
  Evidence of iterative improvement (v1.0 → v1.2+)  

- **A/B Results**  
  Demonstrating improvements in clarity, structure, and usability

## Responsible AI Use

The prompts are designed with governance in mind.

### Key Risks
- hallucination  
- incorrect or incomplete inputs  
- privacy concerns  
- over-reliance on AI  

### Mitigation Strategies
- human review before use  
- strict prompt constraints  
- exclusion of sensitive information  
- validation of input data  
- escalation rules for uncertainty  

AI is positioned as a **support tool**, not a decision-maker.
